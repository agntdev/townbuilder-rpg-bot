# Lightweight RPG Town Builder Bot Design Document

## Summary
This Telegram bot transforms a single group chat into a collaborative RPG town-building experience. Group members passively generate "gold" (from messages) and "food" (from reactions), which can be spent via a `/spend` command to unlock and construct buildings. The bot tracks the town's shared resources and progress, sending notifications to the group when buildings are constructed. Designed for community engagement through passive participation, with future expansion potential for events and inter-group competition.

## Core Entities
- **Town**: Central entity representing the group's shared RPG project. Each group chat has one Town.
- **Resources**: 
  - Gold (generated by messages)
  - Food (generated by reactions)
- **Buildings**: Structures requiring specific resource costs to unlock. Starts with a basic list (e.g., "Town Hall", "Bakery").
- **Group Chat**: The environment where the bot operates, tracked by its unique Telegram ID.

## External Dependencies
- **Telegram Bot API**: 
  - Message tracking (for gold generation)
  - Reaction tracking (for food generation)
  - Command handling (e.g., `/spend`)
  - Group chat identification (via automatic detection)
- **Persistence**: 
  - Town state (resources, unlocked buildings)
  - Group chat ID mapping
  - Building unlock status

## Full Feature List
- [ ] Automatic group detection when bot is added
- [ ] Gold generation: +1 gold per message sent in the group
- [ ] Food generation: +1 food per reaction added to any message
- [ ] `/spend` command to open building selection interface
- [ ] Display list of available buildings with resource costs
- [ ] Deduct resources and construct selected building if sufficient funds
- [ ] Send group notification confirming construction with building name
- [ ] Persist town resources, unlocked buildings, and group chat ID
- [ ] Prevent resource generation from bot messages or reactions
- [ ] Display current resource balances on `/spend` invocation

## Non-Goals
- Inter-group competition or territory mechanics
- Automatic discovery of other groups
- Player-vs-player combat systems
- Real-money transactions
- Complex building interactions beyond construction notifications
- Random events or quests in initial version
